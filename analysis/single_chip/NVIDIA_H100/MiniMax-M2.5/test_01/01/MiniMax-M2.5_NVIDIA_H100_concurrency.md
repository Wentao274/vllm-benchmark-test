## 测试场景
在固定请求数，输入上下文和输出上下文长度下，使用vllm bench serve工具对并发数逐级增加场景的性能基准验证。分析同一芯片同一模型在不同并发级别下的性能指标变化趋势。

**主要采集指标**：

| 指标                  | 单位         | 含义                                 |
|---------------------|------------|------------------------------------|
| TTFT                | ms         | Time To First Token，首 token 延迟     |
| TPOT                | ms/token   | Time Per Output Token，每 token 生成时间 |
| Throughput          | tokens/s   | 系统总吞吐                              |
| QPS                 | requests/s | 请求吞吐                               |
| P50/P95/P99 Latency | ms         | 延迟分位数                              |

## 📝 分析总结

### 1. 吞吐量性能分析

**请求吞吐量 (QPS)**: 随着并发级别增加，QPS持续上升。
低并发(1,2,4)平均 QPS: 0.72 req/s；
中并发(8,10,16,32)平均 QPS: 1.72 req/s；
高并发(64,80,128)平均 QPS: 1.75 req/s；
最高 QPS 出现在 16 并发，达到 1.76 req/s。

**Token总吞吐量**: 最高达到 18428 tok/s (16 并发)。

### 2. 首Token延迟 (TTFT) 分析

TTFT随并发增加显著上升。
低并发平均 P99 TTFT: 909ms；
高并发平均 P99 TTFT: 48381ms；
最高 P99 TTFT 出现在 128 并发，达到 69615ms。

### 3. Token生成时间 (TPOT) 分析

TPOT随并发增加也呈上升趋势。
低并发平均 P99 TPOT: 11.13ms；
高并发平均 P99 TPOT: 21.36ms；
最高 P99 TPOT 出现在 64 并发，达到 21.36ms。

### 4. Token间延迟 (ITL) 分析

ITL随并发增加呈上升趋势。
低并发平均 P99 ITL: 10.55ms；
高并发平均 P99 ITL: 189.49ms；
最高 P99 ITL 出现在 10 并发，达到 190.68ms。

### 5. 综合评估

**吞吐量增长**: 从最低并发到最高并发，QPS增长了 373.0%。
**TTFT延迟恶化**: 高并发相比低并发，TTFT P99增加了 7559.8%。
**TPOT延迟恶化**: 高并发相比低并发，TPOT P99增加了 91.9%。
